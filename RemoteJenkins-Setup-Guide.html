Step 1 — Install Git on Remote Server

Open PowerShell as Administrator on the remote server and run:

Invoke-WebRequest -Uri "https://github.com/git-for-windows/git/releases/download/v2.44.0.windows.1/Git-2.44.0-64-bit.exe" -OutFile "$env:TEMP\git-installer.exe"
Start-Process -FilePath "$env:TEMP\git-installer.exe" -ArgumentList "/VERYSILENT /NORESTART /NOCANCEL /SP-" -Wait

Verify (open new CMD):

git --version

Expected: git version 2.44.0.windows.1

Step 2 — Install .NET 8 SDK on Remote Server

In the same Admin PowerShell:

Invoke-WebRequest -Uri "https://download.microsoft.com/download/dotnet/8.0/8.0.408/dotnet-sdk-8.0.408-win-x64.exe" -OutFile "$env:TEMP\dotnet-sdk8-installer.exe"
Start-Process -FilePath "$env:TEMP\dotnet-sdk8-installer.exe" -ArgumentList "/install /quiet /norestart" -Wait

Verify (open new CMD):

dotnet --version

Expected: 8.0.x

Step 3 — Add Git and .NET to Windows System PATH

Run in Admin PowerShell — this ensures Jenkins service can find both tools:

$currentPath = [System.Environment]::GetEnvironmentVariable("PATH", "Machine")
$toAdd = @("C:\Program Files\Git\bin", "C:\Program Files\dotnet")
foreach ($p in $toAdd) {
    if ($currentPath -notlike "*$p*") {
        $currentPath = "$currentPath;$p"
        Write-Host "[ADDED] $p"
    }
}
[System.Environment]::SetEnvironmentVariable("PATH", $currentPath, "Machine")
Restart-Service -Name "Jenkins"
Write-Host "Done — wait 20 seconds then open http://localhost:8080"

  2. Login with Jenkins credentials
  3. Go to: Manage Jenkins → Plugins → Available plugins
  4. Search and install each plugin below (tick checkbox → click Install):

Step 5 — Configure Allure Commandline Tool

  1. Go to: Manage Jenkins → Tools
  2. Scroll to Allure Commandline → click Add Allure Commandline
  3. Set:
    • Name: allure
    • Install automatically: checked
    • Version: select latest (e.g. 2.27.0)
  4. Click Save

Step 6 — Create the Pipeline Job

  1. Go to: Dashboard → New Item
  2. Item name: WillScot-Automation

  1. On the job page click Build Now
  2. Click the build number → Console Output to watch progress
  3. Wait ~15 minutes for full run (restore + build + Playwright download + tests)
Expected final line: Finished: SUCCESS

Step 9 — Verify Auto-Trigger on Git Push

On the local developer machine, push any change:

git commit --allow-empty -m "ci: verify remote Jenkins auto-trigger"
git push origin main

Wait up to 2 minutes → check the job page on remote Jenkins — a new build should appear automatically triggered by SCM change.

Confirm polling is active: Job page → Git Polling Log (left sidebar)

Build not auto-triggeringCheck Git Polling Log; verify branch is */main not */master
Allure report step UNSTABLEExpected — Jenkinsfile wraps it in catchError; test results still published
